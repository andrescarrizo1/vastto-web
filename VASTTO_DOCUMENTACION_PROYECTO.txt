1. ¿QUÉ ES VASTTO?
-------------------
VASTTO (vastto.com.ar) es un sitio web de afiliados de Mercado Libre Argentina.
El modelo de negocio es el siguiente:
- El sitio muestra productos de tecnología con contenido original (guías, 
  comparativas, reseñas).
- Cada producto tiene un link que lleva a Mercado Libre.
- Ese link contiene un código de referido (matt_tool=caan170047).
- Cuando alguien compra (directa o indirectamente), el dueño gana una comisión.
- El programa se llama "Mercado Libre Afiliados".

Dominio: vastto.com.ar
Hosting: Cloudflare Pages (deploy automático desde GitHub)
Repositorio: https://github.com/andrescarrizo1/vastto-web
Framework: Astro (generador de sitios estáticos)
Código de afiliado: caan170047 (en src/utils/affiliate.ts)


2. ESTRATEGIA SEO — EL CONCEPTO CLAVE
--------------------------------------
PROBLEMA ORIGINAL:
La web estaba armada como un clon de Mercado Libre — listados de productos con 
precios. Esto NUNCA posiciona en Google porque:
- ML tiene millones de páginas y décadas de autoridad de dominio.
- Google nunca va a posicionar una web chica por "auriculares bluetooth" o 
  "celulares baratos" cuando ML ya cubre esas keywords.
- El sitio solo tenía 5 páginas indexables — insuficiente para que Google 
  considere el sitio como una autoridad en algo.

SOLUCIÓN IMPLEMENTADA (estilo Romualds Founds):
Pivotar a contenido de LONG-TAIL — escribir artículos/guías que responden 
preguntas específicas que la gente busca y que Mercado Libre NO cubre.

Ejemplos de long-tail keywords que atacamos:
- "mejor celular gama media argentina 2026" (ML no tiene comparativas)
- "samsung vs motorola cual es mejor" (ML no compara marcas)
- "mejores smartwatch baratos argentina" (ML no tiene guías de compra)
- "auriculares para viajar en colectivo" (nicho muy específico)
- "accesorios indispensables para celular" (ML no recomienda)

La clave: el usuario busca esa pregunta → llega a nuestra guía → lee el 
contenido → hace click en el producto que le gustó → va a ML con nuestro link 
de afiliado → compra → nosotros ganamos la comisión.


3. CAMBIOS REALIZADOS (27/03/2026)
-----------------------------------

A) TECHNICAL SEO FIXES:
   
   - src/consts.ts
     ANTES: SITE_TITLE = "Vastto — Los mejores productos en Mercado Libre"
     AHORA: SITE_TITLE = "VASTTO"
     POR QUÉ: El título se duplicaba en el <title> de cada página, quedando
     algo como "VASTTO | Auriculares | Vastto — Los mejores...". Ahora queda
     limpio: "Auriculares Gamer Baratos | VASTTO".

   - src/components/BaseHead.astro
     * Arreglado el <title> para no duplicar nombre del sitio
     * Agregado hreflang="es-AR" para que Google sepa que el sitio es argentino
     * Agregado geo.region=AR y geo.placename=Argentina
     * Mejorado el schema de WebSite
     POR QUÉ: Google necesita señales claras de que el contenido es para
     Argentina. Sin hreflang, podría mostrar la web a usuarios de España o
     México que no compran en ML Argentina.

   - src/styles/global.css
     * Eliminadas variables CSS de dark mode (el sitio es light mode)
     * Arreglados colores de headings (eran blancos sobre fondo blanco)
     POR QUÉ: Había conflicto entre los estilos globales (dark) y los estilos
     inline de las páginas (light). CSS innecesario = bytes desperdiciados.

   - public/robots.txt
     ANTES: Bloqueaba /search/ y /checkout/ (rutas que NO existen)
     AHORA: Solo "Allow: /" + referencia al sitemap
     POR QUÉ: Reglas de bloqueo para rutas inexistentes confunden a Google.

   - public/sitemap.xml
     ANTES: 5 URLs estáticas sin lastmod
     AHORA: 13 URLs con lastmod, changefreq y priority
     POR QUÉ: Google necesita el sitemap para descubrir páginas nuevas.
     Sin lastmod, no sabe cuándo se actualizó cada página.

4. SISTEMA DE AFILIADOS — CÓMO FUNCIONAN LOS LINKS
---------------------------------------------------
Archivo clave: src/utils/affiliate.ts

Contiene:
- AFFILIATE_ID = 'caan170047'
- Función addAffiliateParam(url) que agrega ?matt_tool=caan170047 a cualquier
  URL de Mercado Libre.

El componente ProductCard.astro usa esta función automáticamente. Cualquier
producto que se agregue con una URL de ML tendrá el código de afiliado.

IMPORTANTE: Si se cambia de cuenta de afiliado, solo hay que cambiar el ID en
affiliate.ts y todo el sitio se actualiza automáticamente.


5. STACK TÉCNICO
----------------
- Framework: Astro 4.x (generador estático)
- Hosting: Cloudflare Pages (auto-deploy desde GitHub main branch)
- CSS: Vanilla CSS con variables custom (sin Tailwind)
- Fonts: Inter (Google Fonts)
- Sitemap: Estático en public/sitemap.xml
- Build: npm run build → genera HTML estático en dist/
- Dev: npm run dev → servidor local en localhost:4321
